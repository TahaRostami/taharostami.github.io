    Some Notes About FaRM article #

    Fault Revealing Mutant Selection Prblem Definition #

    Thierry Titcheu Chekam et. al. defined a fault revealing mutant selection problem as:

    Given: A set of mutants M for program P.

    Problem: Subset selection. Select a subset of mutants, S ∈ M, such that F (S) = F (M) and (∀m ∈ S), (F (S − {m}) = F (M)).

    S represents a subset of M; F (X) represents the number of faults in P that are revealed by the test suites that kill all the mutants of the set X. In practice, the challenge is to approximate well S, statically and prior to any test execution, by finding a relatively good trade-off between the number of selected mutants (to minimise) and the number of faults revealed by their killing (to maximize).

    Their Implementaion #

    For this purpose, they used C programs prepered in the Codeflaws dataset. Codeflaws tree structure as follows:

    1. Number of problems defined for online programming competition which we called each problem as project (problem = project).
    2. Number of problem might solved with different programmers, this means we have one or more implementations for each problem. we called each of this program as implementaion (program = implementaion).
    3. for each implementaion we have two versions, one is sumbited program whcih rejected because of existence of bug(s) in it. another is submited program which accepted because of the test suite did not find any bug in it. In the absence of the oracles for the programs, they use accepted one behavior as expected behavior and all of their mutation performed on the rejected one.

    They produce Mutants for each C programs using Mart. Mart get a C program, and perform mutation on the llvm file of the C program. In particular they said that all of their analysis are based on llvm codes (not C codes).

    Codeflaws also contains some test cases for each problem, but as they might not be strong enough, they also produce strong test inputs for each of the problem using KLEE.

    They remove equivalent mutants that identified by the TCE.

    After that, they execute mutation analysis process and using the matrix execution calculate utility for each of the mutants. they consider utility of the mutant as #number of test cases that kill the mutant and reveal faults / #number of test cases that kill the mutants. If mutant did not kill by any test cases they consider it as equivalent mutant and set it’s utility as -1. They also remove from the dataset, trivial mutants which are mutants that many test cases can kill them.

    For reminder of the mutants, they extract feature values. and then made the dataset available.

    Note their experiment is performed on the mention dataset (without any change). According our communication with them, they verified that the following pseudocode is what they perform in experiment 1 of their work:

    01. dataset <- load Codeflaws dataset
02. dataset['coupling-weight'] <- 'fault-rev' if dataset['coupling-weight']=1 else 'not-fault-rev'
03. parts <- randomly splitting the dataset into 10 parts such that all implementations of the same problem appear exactly in one part
04. for (each part in parts)
05.    testx,y <- part
06.    Trainx,y <- parts \ part
07.    trainx,y <- Oversample(trainx,y)
08.    model <-StochasticGradientBoostingClassifier(depth=5,tree=1000).FitModel(trainx,y)
09.    preds <- model.PredProb(testx,y).sort()
10.    select 05% top from preds and then report precision, recall, f1
11.    select 10% top from preds and then report precision, recall, f1
12.    select 20% top from preds and then report precision, recall, f1
13.    report AUC

    They are consider confusin matrix as follows:

    Predicted fault-rev Predicted not fault-rev
    Actually fault-rev TP FN
    Actually not fault-rev FP TN

    according to the above table we know:

    \( \boxed{\text{Precision }=\frac {number {\space} of {\space}fault-rev {\space}mutants{\space} which{\space} predicted{\space} as {\space} fault-rev} {number {\space} of {\space}mutants{\space} predicted{\space} as{\space} fault-rev}} \) \( \boxed{\text{Recall }=\frac {number {\space} of {\space}fault-rev {\space}mutants{\space} which{\space} predicted{\space} as {\space} fault-rev} {number {\space} of {\space}actual{\space} fault-rev{\space} mutants}} \)

    They report their results of the experiment 1 as follows:

    AUC:62.04% Precision Recall F1
    5% top mutants 5.7% 12.8% 7.8%
    10% to mutants 4.9% 22% 8%
    20% to mutants 3.9% 35.1% 7%
    NOTE
    In this application recall is seem more appealing that precision.
