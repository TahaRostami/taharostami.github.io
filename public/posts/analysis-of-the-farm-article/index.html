    From the above results we know:

    1. 416,929 mutants in the dataset are exactly equivalents (23.7% from the total mutants).

    2. For the 3,048‬ not-fault revealing mutants, exists at least one fault-revealing mutants with the excatly same embedding vector (i.e. for 8.6% of the fault-rev mutants the same instance exists with the label of not-fault-rev).

    From the Observation 2 we know there exists some instances like below:

    Feature1 Feature2 … FeatureX target
    V1 V2 … VX fault-rev
    V1 V2 … VX not-fault-rev

    We argue that there are probably many instances that are very, very close to each other.

    We also claim for an arbitrary fault-rev mutant with the embedding vector V, we can generate unlimited number of vector close to it which they are not fault revaling mutants.

    But some challenges exists:

    1. It seems density based and transductive clustering approaches like DBScan best suite for our purpose but theie implementaion doesn’t support predict function for new data.

    1. What distance measures must we use? All of our features are between [0,1] and our dataset dimension is high so It seems Euclidean distance doesn’t suite.
